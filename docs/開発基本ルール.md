# Pixel Sync Box 開発基本ルール

最終更新: 2026-06-18

---

# 開発理念

Pixel Sync Box は

「大量の写真・動画を、人の操作なしでGoogleフォトへ安全・確実にバックアップする」

ことを目的としたプロジェクトである。

コードの美しさよりも、

**確実に動くこと**

を最優先とする。

---

# 基本方針

## 1. まず動かす

最初から完璧な設計は目指さない。

まず完成させる。

動作確認後に改善する。

---

## 2. リファクタリングは後

優先順位

```
動く

↓

安定する

↓

速くする

↓

きれいにする
```

性能改善は完成後に行う。

---

## 3. モジュール単位で完成

一度に全部作らない。

完成させる順番

```
Scanner

↓

Database

↓

Queue

↓

Upload

↓

UI

↓

Worker

↓

App
```

完成したモジュールは基本的に触らない。

---

## 4. 小さくテストする

モジュール完成

↓

単体テスト

↓

問題修正

↓

Appへ統合

↓

全体テスト

---

# Pythonファイルのルール

すべてのPythonファイル先頭に必ず記載する。

```python
# ============================================
# FILE: filename.py
# VERSION: x.x.x
# UPDATED: YYYY-MM-DD
# ============================================
```

例

```python
# ============================================
# FILE: pixel_sync/ui.py
# VERSION: 1.2.0
# UPDATED: 2026-06-18
# ============================================
```

変更したら必ずVERSIONを更新する。

---

# コード提供ルール

ChatGPTやCursorでコードを作成する場合

## 小変更

10〜20行程度

→ bash差分で修正

例

```bash
sed -i '' '45s/old/new/' pixel_sync/ui.py
```

---

## 大変更

20行以上

または

新規ファイル

↓

全文を提示する。

中途半端な差分は作らない。

# 命名規則

関数名

```
snake_case
```

例

```
get_backup_status()

prepare_ui()

wait_backup_complete()
```

定数

```
UPPER_CASE
```

例

```
UI_XML
ADB_TIMEOUT
```

# UI解析方針

GoogleフォトAPIは使用しない。

ADB + UIAutomatorで画面を解析する。

判定対象

```
バックアップ完了

バックアップ中

アップロード中

残り○件

チュートリアル

ロック画面
```

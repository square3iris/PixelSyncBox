# Pixel Sync Box 開発基本ルール

---

# 開発理念

Pixel Sync Box は

「大量の写真・動画を、人の操作なしでGoogleフォトへ安全・確実にバックアップする」

ことを目的としたプロジェクトである。

コードの美しさよりも

**確実に動くこと**

を最優先とする。

---

# 基本方針

## 1. まず動かす

最初から完璧な設計は目指さない。

まず完成させる。

動作確認後に改善する。

---

## 2. リファクタリングは後

優先順位

```text
動く

↓

安定する

↓

速くする

↓

きれいにする
```

性能改善は完成後に行う。

---

## 3. モジュール単位で完成

一度に全部作らない。

完成順

```text
Scanner

↓

Database

↓

Queue

↓

Upload

↓

UI

↓

Worker

↓

App
```

完成したモジュールは極力触らない。

---

## 4. 小さくテストする

```text
モジュール完成

↓

単体テスト

↓

問題修正

↓

Appへ統合

↓

全体テスト
```

---

# ChatGPT共同開発ルール

## 1. 極力コピペだけで実行できる指示にする

説明より先に実行可能なコマンドを提示する。

例

```bash
grep ...
sed ...
python3 ...
```

---

## 2. 少量修正は bash で変換する

数行修正のために手編集は極力行わない。

優先順位

```text
1. bash変換
2. python変換スクリプト
3. 手編集
```

---

## 3. 50行を超える変更はファイル丸ごと置換

形式

```bash
cat > file.py <<'PY'
...
PY
```

部分差分より安全性を優先する。

---

## 4. 100行を超えたら分割を検討

厳密な制限ではない。

判断基準

- 読みやすさ
- 保守性
- 責務分離

必要なら100行を超えてもよい。

---

## 5. 1ファイルずつ修正する

禁止

- 複数ファイル同時修正
- 一括リファクタ

推奨

```text
修正

↓

確認

↓

次ファイル
```

---

## 6. 毎回動作確認する

```text
修正

↓

確認コマンド

↓

結果確認

↓

次へ進む
```

---

## 7. 修正前に必ず現状確認する

いきなり修正しない。

例

```bash
grep ...
sed ...
ls ...
```

現状確認後に修正する。

---

## 8. 設計を確認してから実装する

まず

- なぜ修正するか
- どのファイルを触るか
- 影響範囲

を明確にする。

---

## 9. 動くものを優先する

過度な設計より

```text
動く

↓

確認

↓

改善
```

を優先する。

---

## 10. リファクタと機能追加を同時にやらない

順序

```text
機能追加

↓

動作確認

↓

コミット

↓

リファクタ
```

# Pythonファイルルール

全Pythonファイル先頭に記載する。

```python
# ============================================
# FILE: filename.py
# VERSION: x.x.x
# UPDATED: YYYY-MM-DD
# ============================================
```

変更時は VERSION を更新する。

---

# 命名規則

## 関数

snake_case

例

```python
get_backup_status()
wait_backup_complete()
```

---

## 定数

UPPER_CASE

例

```python
ADB_TIMEOUT
UPLOAD_THREADS
```

---

# 設定管理ルール

環境依存値はコードへ直書きしない。

設定は

```text
pixel_sync/settings.py
```

へ集約する。

例

- NASパス
- DBパス
- Pixelパス
- 言語設定
- スレッド数
- タイムアウト

---

# ログ管理ルール

重要処理は Logger を使用する。

対象

- 起動
- 終了
- エラー
- リトライ
- DB処理
- Upload
- Delete
- Backup監視

---

# UI解析方針

GoogleフォトAPIは使用しない。

ADB + UIAutomator を使用する。

判定対象

```text
バックアップ完了
バックアップ中
アップロード中
残り件数
チュートリアル
ロック画面
```
